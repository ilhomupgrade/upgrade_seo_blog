# RAG-системы: как научить ИИ работать с вашими документами

Представьте: у вас куча документов - ТЗ на 40 страниц, инструкции, договоры. Сотрудник спрашивает о сроках, а ChatGPT выдает галлюцинации из головы. Знакомо? RAG (Retrieval Augmented Generation) решает это. Технология подключает языковую модель к вашей базе знаний. ИИ ищет релевантные фрагменты и генерирует точные ответы на основе ваших данных.[1][2]

RAG превращает хаос документов в умного помощника. Новички быстро находят правила компании, менеджеры - детали контрактов. Без дообучения модели. Просто и эффективно.

## Что такое RAG и зачем оно нужно

RAG - это мост между LLM и внешними данными. Обычная нейросеть помнит только то, на чем обучалась. Ваши свежие документы? Она их не знает. RAG добавляет шаг поиска: модель находит нужное и вставляет в промпт.[3][5]

**Ключевые плюсы RAG:**
- Точность: ответы на основе фактов, а не выдумок.
- Актуальность: работает с новыми данными без переобучения.
- Экономия: не нужно тратить миллионы на fine-tuning.
- Контекст: ИИ понимает специфику вашей компании.

Пример: в банке RAG анализирует политики и отвечает клиентам по кредитам. Ошибок меньше, скорость выше.[1]

Без RAG ИИ импровизирует, как стендапер. С RAG - цитирует смысл из ваших файлов.[2]

## Как работает RAG под капотом

RAG - это пайплайн из пяти шагов. Разберем по порядку.[2][4]

### Шаг 1: Загрузка и подготовка документов

Берете PDF, DOCX, TXT. Конвертируете в единый формат - часто Markdown. Используйте Docling или LangChain для извлечения текста.[1][7]

Очистите шум: уберите футеры, таблицы, если не нужны.[7]

### Шаг 2: Чанкование - разбиваем на куски

ИИ не жрет целые книги. Окно контекста - 300-500 слов. Делите текст на **чанки** - логические фрагменты.[4][8]

Хороший чанк:
- Отвечает на один вопрос.
- 300-500 слов.
- С заголовком.
- Независимый.

LangChain с RecursiveCharacterTextSplitter режет по предложениям и абзацам.[4]

Пример:
- Плохой чанк: вся глава.
- Хороший: "Сроки оплаты: 30 дней с даты отгрузки".

### Шаг 3: Эмбеддинги - превращаем в векторы

Каждый чанк -> вектор. Это числовое представление смысла. Модели типа sentence-transformers или OpenAI embeddings.[3]

Вектор отражает семантику: "сроки платежа" близок к "оплата по договору".[3]

### Шаг 4: Векторная база - хранилище знаний

Векторы в Qdrant, Pinecone или FAISS. Поиск по сходству - cosine similarity.[3][5]

Запрос -> эмбеддинг -> топ-5 похожих чанков.

### Шаг 5: Генерация ответа

LLM получает промпт: "Вопрос: [query]. Контекст: [чанки]. Ответь точно".[1]

ИИ синтезирует: цитирует, объясняет, добавляет ссылки.

## Продвинутые фишки RAG

- **Дообучение:** На формате "вопрос + документы" для стиля ответов.[1]
- **Гибридный поиск:** Ключевые слова + векторы.
- **Рерангинг:** Переранжировка чанков для топ-релевантности.
- **Мультимодал:** Изображения, таблицы в чанки.[7]

В Yandex Cloud: Files API для загрузки, готовый пайплайн.[1]

Пример в бизнесе: HR-чат. Новичок: "Как оформить отпуск?" RAG тянет из политики и отвечает с ссылкой.

## Проблемы и как их фиксить

- Плохие чанки: смысл теряется. Решение: overlap 20%.
- Неточные эмбеддинги: смените модель на русскоязычную (LaBSE).
- Галлюцинации: добавьте в промпт "Цитируй только контекст".
- Масштаб: для 1000+ доков - облако вроде Yandex или Pinecone.[1][3]
